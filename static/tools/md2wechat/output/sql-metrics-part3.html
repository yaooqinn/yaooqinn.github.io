Spark SQL Metrics 三部曲的第三部分（终章）

DataSource V2 CustomMetric API

如果你正在构建自定义连接器，需要暴露连接器特有的数据——例如读取字节数、缓存命中率或限流次数——从 Spark 3.2 开始，DataSource V2 API 提供了干净的扩展点。

接口层次结构

CustomMetric——在连接器中定义一次，描述指标是什么：

  • name()、description()——名称和描述
  • aggregateTaskMetrics(long[])——你决定如何聚合各任务的值
  • 必须有无参构造函数（Spark 通过反射实例化）

CustomTaskMetric——由 PartitionReader 在 Executor 上报告，包含 name() + value()。

内置基类：

类聚合逻辑输出格式
CustomSumMetric求和String.valueOf(sum)
CustomAvgMetric平均值#0.000 格式

三步实现自定义指标

第一步：定义指标类

public class MyBytesReadMetric extends CustomSumMetric {
    @Override public String name() { return "myBytesRead"; }
    @Override public String description() { return "bytes read from my source"; }
}

第二步：在 Scan 中注册

@Override
public CustomMetric[] supportedCustomMetrics() {
    return new CustomMetric[] { new MyBytesReadMetric() };
}

第三步：在 PartitionReader 中报告（每 100 行调用一次）

@Override
public CustomTaskMetric[] currentMetricsValues() {
    return new CustomTaskMetric[] {
        new CustomTaskMetric() {
            public String name() { return "myBytesRead"; }
            public long value() { return bytesReadSoFar; }
        }
    };
}

就这样——自定义指标将在 Spark UI 中与内置指标一起显示。写入端也支持，通过 BatchWrite.supportedCustomMetrics() 和 DataWriter.currentMetricsValues()。

UI 中的指标渲染

渲染管道

SQLAppStatusStore.executionMetrics(id)
    → Map[accumulatorId → formatted String]

ExecutionPage.planVisualization()
    → graph.makeDotFile(metrics)        # 紧凑 DOT 标签
    → graph.makeNodeDetailsJson(metrics) # 完整指标 JSON

spark-sql-viz.js
    → renderPlanViz()                    # dagre-d3 图形
    → getNodeDetails()                   # 解析 JSON
    → updateDetailsPanel()              # 点击侧边面板
    → rerenderWithDetailedLabels()       # 内联模式

紧凑模式 vs 详细模式

  • 紧凑模式（默认）：节点仅显示算子名，点击打开侧边面板查看指标
  • 详细模式（复选框）：10px 字号内联显示在节点内
  • Stage/Task 切换：最大值旁添加 (stage X: task Y) 标注

REST API

主要端点：

GET /api/v1/applications/{appId}/sql/{executionId}
    ?details=true&planDescription=true

关键要点：Metric 仅是 {name: String, value: String}——返回格式化后的显示字符串，而非原始数值。

通过 spark-history-cli 访问

# 结构化 JSON 输出
spark-history-cli --json -a <app> sql          # 列出所有 SQL 执行
spark-history-cli --json -a <app> sql <id>     # 单次执行及其指标
spark-history-cli -a <app> sql-plan <id> --view final  # AQE 后计划

实用示例

用 Spark Listener 捕获原始指标值

spark.listenerManager.register(new QueryExecutionListener {
  override def onSuccess(funcName: String, qe: QueryExecution,
      durationNs: Long): Unit = {
    val metrics = qe.executedPlan.collectLeaves().flatMap(_.metrics)
    metrics.foreach { case (name, metric) =>
      println(s"$name: ${metric.value}")  // 原始 Long 值！
    }
  }
  override def onFailure(...): Unit = {}
})

这是获取未经格式化和四舍五入的原始 Long 指标值的唯一方式。REST API 和 UI 都返回格式化后的字符串。
